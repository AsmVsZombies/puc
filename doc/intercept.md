# 拦截指令语法

`puc intercept "<command>"`（以及 MCP 工具 `puc_intercept`）把一段**分号分隔**的指令串
喂给一个有状态的解析器，逐条执行并各输出一行结果。

```sh
puc intercept "pe; wave 1 400 800; delay 8.8"
# delay row=1,5 col=8.8 garg_rows=[1,2,5,6] intercept=212~230 eat=410 iceable=445
```

## 通用规则

- 用分号 `;` 分隔多条指令；**同一次调用内**分号前后的状态（场合、用冰/激活时机）保留。
- 解析前整串转小写，首尾空格忽略，空指令段跳过。
- 从第一条失败的指令处终止；CLI 进程退出码为 `1`，MCP 返回 `isError` 结果。
- 初始默认：场合 `pe`（后院）、激活时机 `318`、无用冰。
- 数值参数支持算术表达式，例如 `8.8`、`445+200`、`9-1/16`。
- 列、坐标均为 1 基编号；时间单位为厘秒（cs），列为炮单位（1 列 = 80 px）。
- 巨人 x 范围、行号列表、坐标对等用英文逗号 `,` 分隔（中文逗号 `，` 也接受）。

## 指令一览

| 指令 | 说明 |
| --- | --- |
| `de` / `ne` | 设置前院（DE），静默 |
| `pe` / `fe` | 设置后院（PE，默认），静默 |
| `re` / `me` | 设置屋顶（RE），静默 |
| `wave` | 查询当前用冰、激活时机，输出 `wave ice=[…] cob=… garg_x=[…,…]` |
| `wave 冰时机.. 激活时机` | 设置用冰、激活时机（用冰时机可为 0 个或多个），静默 |
| `delay 炮列 (炮尾列)` | 计算可拦区间、最早啃食、最早可冰 |
| `delay1/2/3 …` | 同上，强制按拦截一/两/三行计算 |
| `delay 炮行 炮列 (炮尾列) > 巨人行 (巨人x范围) (u/i)` | 计算炮拦截特定巨人 |
| `doom 核行 核列 (> 巨人行 (巨人x范围) (u/i))` | 计算核武拦截特定巨人 |
| `hit (炮尾列) (延迟)` | 计算刚好全伤巨人的炮落点 |
| `nohit (炮尾列) (延迟)` | 计算刚好不伤巨人的炮落点 |
| `max 炮行 炮列范围 (炮尾列) > 巨人行 (巨人x范围) (u/i)` | 寻找无伤拦截可延迟最多的炮落点列 |
| `imp 巨人x` | 计算该巨人投掷的小鬼 x 坐标范围 |
| `imp garg 小鬼x（或 x1,x2）` | 计算投掷该坐标 / 区间小鬼的巨人 x 范围 |

> 括号 `( )` 内为可选参数。屋顶场合需要额外的「炮尾列」参数。

## 取值范围

| 参数 | 范围 |
| --- | --- |
| 炮行 / 核行 | 场地全部行（DE、RE 为 `1..5`；PE 为 `1..6`） |
| 炮列（落点列） | `0 ≤ 列 < 10`；`列 × 80` 需为整数，否则给出警告并自动修正 |
| 炮尾列（屋顶） | `1..8` |
| 核列 | `1..9` |
| 巨人所在行 | 对应炮 / 核可命中的巨人行，超范围的行会被过滤 |
| 巨人 x 范围 | `min,max`；`min` 需大于下界、`max` 不超过上界 |
| 用冰标记 | `u` = 不计冰减速；`i` = 计冰减速；省略时沿用当前 `wave` 状态 |

## 指令详解

### 场合 `de` / `pe` / `re`

切换前院 / 后院 / 屋顶，影响行数、炮飞行距离与巨人 x 范围。同义写法
`ne` / `fe` / `me`。成功时静默无输出。

### `wave`

- 无参数：查询当前用冰、激活时机及对应巨人 x 范围。
- `wave 冰时机.. 激活时机`：最后一个值为激活（发炮）时机，前面 0 个或多个为用冰时机。
  例：`wave 800`（无冰）、`wave 1 400 800`（两次用冰 + 激活）。激活时机须 `≥ 0`。

### `delay` / `delay1` / `delay2` / `delay3`

计算炮拦截的可拦区间、最早啃食时机、最早可冰时机。

- 简写形式 `delay 炮列 (炮尾列)`：按当前场合默认的命中行 / 拦截行数计算。
  `delay1/2/3` 强制拦截 1 / 2 / 3 行。
  - 后院 / 前院：`delay 8.8`
  - 屋顶：`delay 3.5 4`（追加炮尾列）
- 完整形式 `delay 炮行 炮列 (炮尾列) > 巨人行 (巨人x范围) (u/i)`：指定炮位与目标巨人。
  - 后院 / 前院：`delay 1 8.8 > 1,2 700,800 u`
  - 屋顶：`delay 1 3.5 4 > 1,2 700,800 u`
  - `>` 之后依次为：巨人所在行（逗号分隔）、可选巨人 x 范围、可选用冰标记。

### `doom`

`doom 核行 核列 (> 巨人行 (巨人x范围) (u/i))` 计算核武（玉米核弹）拦截特定巨人。
`>` 之后的巨人参数可省略，省略时使用该核列默认覆盖的巨人行。结果按植物拦截位移调整。

例：`doom 3 5`、`doom 3 5 > 2,3 700,800 i`。

### `hit` / `nohit`

计算「刚好全伤」/「刚好不伤」最靠后行巨人的炮落点。

- 后院 / 前院：`hit (延迟)`、`nohit (延迟)` —— 可选延迟会临时叠加到激活时机上。
- 屋顶：`hit 炮尾列 (延迟)`、`nohit 炮尾列 (延迟)` —— 炮尾列必填。

`hit` 取巨人 x 范围上界（最靠右、最难全伤），`nohit` 取下界（最靠左、最易擦伤）。

### `max`

`max 炮行 炮列范围 (炮尾列) > 巨人行 (巨人x范围) (u/i)` 在给定落点列区间内，
寻找能**无伤拦截且可延迟最多**的炮落点列，并输出该列对应的可拦区间。

- 炮列范围写作 `min,max`，例：`max 1 8,9 > 1,2 700,800 u`。
- 屋顶在炮列范围后追加炮尾列：`max 1 3,4 4 > 1,2 700,800 u`。
- 若区间内不存在无伤落点，输出对应提示。

### `imp`

- `imp 巨人x`：给定巨人 x 坐标（须 `> 400`），输出其投掷小鬼的 x 坐标范围
  （随机量取整 0 与取整 100 两种边界）。
- `imp garg 小鬼x`：给定单个小鬼 x，反推可投出该坐标的巨人 x 范围。
- `imp garg x1,x2`：给定小鬼 x 区间，输出并集巨人 x 范围。超界会被裁剪到合法区间。

## 输出约定

- 每个非静默指令输出**一行**，格式为 `命令 key1=value1 key2=value2 …`。
- 拦截区间记为 `intercept=A~B`；无解记为 `intercept=cannot`；可达上限记为 `intercept=A+`。
  存在有伤区间时附加 `unsafe=A~B`。
- 警告（如 `cannot hit all gargs at this tick`、`hit col × 80 not integer`）输出至诊断流
  （CLI 为 stderr，MCP 归入结果文本的 `--- warnings ---` 段）。
- 错误以 `error:` 起首；CLI 退出码为 `1`，MCP 返回 `isError`。
